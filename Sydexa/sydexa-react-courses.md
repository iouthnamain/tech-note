- Mã khóa học: `RCT_301`

  - "React Nâng Cao: Chuyên Sâu với Tối Ưu Hiệu Năng"
- Câu định vị trên trang Sydexa nhấn mạnh đây không phải một khóa React thông thường (định hướng nâng cao, thực chiến, tập trung tối ưu hiệu năng)
- Trạng thái trên nền tảng học: trang thông tin công khai có thể xem được, nhưng ghi danh có thể bị giới hạn theo từng đợt.

#### Hình thức và lịch học

- Hình thức: Online, mentor livestream/share-screen trực tiếp
- Video ghi hình: Có bản ghi sau mỗi buổi
- Thời lượng: 8 tuần (~2 tháng)
- Số buổi học: 16 buổi
- Thời lượng mỗi buổi: 90+ phút/buổi
- Khung giờ thể hiện trong ảnh chụp trang khóa học: Thứ 4 và Thứ 7, 8:00 PM-9:30 PM (giờ Việt Nam)
- Ngày khai giảng xuất hiện trong một ảnh chụp trang: `22/11/2025` (thông tin đợt cũ có thể vẫn hiển thị trên trang giới thiệu khóa học)

#### Hồ sơ học viên phù hợp

- Phù hợp cao:
  - Lập trình viên React có khoảng 6-12 tháng kinh nghiệm thực tế
  - Đã nắm Components, Props, State và các hook cốt lõi (`useState`, `useEffect`)
  - Thành thạo JavaScript hiện đại (`ES6+`, `Promise`, `async/await`)
  - Muốn tiến tới kỳ vọng năng lực frontend cấp Senior
- Không phù hợp:
  - Người hoàn toàn mới với lập trình
  - Người chưa từng xây dựng một ứng dụng React hoàn chỉnh

#### Kỹ năng đầu ra (tổng hợp từ mô tả chính thức)

- Hiểu sâu React internals:
  - Virtual DOM
  - Reconciliation
  - React Fiber
  - Scheduler / concurrent rendering concepts
- Năng lực performance engineering:
  - Phát hiện và giảm các lần re-render không cần thiết
  - Đo lường bằng công cụ (ví dụ: React Profiler, Lighthouse)
  - Virtualization danh sách lớn (`react-window`, `react-virtual`)
  - Code splitting, lazy load, dynamic import, phân tích bundle-size
- Kiến trúc state và data:
  - Pattern global state (Redux / Zustand)
  - Quản lý server-state với React Query
  - Caching, background refetching, invalidation theo best practice
- Kiến trúc form/UI:
  - Controlled vs uncontrolled components
  - Formik + schema validation (`zod` / `yup`)
  - Thực hành UI libraries và accessibility

## Giáo Trình Theo Từng Tuần (Chi Tiết)

> Nội dung bên dưới được lấy từ phần "Nội dung khóa học" công khai trên trang khóa học của Sydexa.

### Tuần 1 - JavaScript Nâng Cao + Nền Tảng React

- Nền tảng cách web vận hành
- Bất đồng bộ, Event Loop, Microtasks
- Khái niệm SPA vs MPA
- Cơ bản kiến trúc và internals của React
- Virtual DOM + Reconciliation
- So sánh sync rendering và concurrent rendering (khái niệm React 18)
- React Fiber + scheduler
- Đào sâu cách dùng hook (`useMemo`, `useCallback`, `useEffect`, `useImperativeHandle`, v.v.)
- Bài tập thực hành: thành thạo hook cơ bản + nâng cao

### Tuần 2 - Tối Ưu Re-render

- Theo dõi và chẩn đoán re-render
- Kỹ thuật tối ưu rendering bằng hook
- Công cụ đo hiệu năng: React Profiler, Lighthouse (và công cụ tương tự)
- Cách dùng `memo`/HOC
- Custom hooks cơ bản + nâng cao
- Bài tập thực hành: render bảng 10.000 items, tách child component để tối ưu

### Tuần 3 - Infinite Loading và Virtualization

- Đào sâu các hướng tiếp cận list virtualization
- `react-window` và `react-virtual`
- Trade-off hiệu năng giữa tải ban đầu và tải thêm
- Sử dụng `IntersectionObserver` API
- Bài tập thực hành: áp dụng virtualization cho bài toán danh sách lớn của Tuần 2

### Tuần 4 - Quản Lý Store + Best Practice Tích Hợp API

- Lựa chọn global store: Redux / Zustand
- Thiết lập tầng API (`axios` được nêu trong giáo trình)
- React Query ở mức cốt lõi:
  - Fetching
  - Mutation
  - Cache handling
  - Background refetch
  - Invalidation
- Bài tập thực hành: tích hợp API với React Query + Zustand

### Tuần 5 - Form Quy Mô Lớn

- Controlled vs uncontrolled components
- Quy trình xây dựng form bằng Formik
- Validation với `zod` / `yup`
- Bài tập thực hành: xây dựng form dạng bảng câu hỏi lớn kèm validation

### Tuần 6 - Routing + UI Libraries

- React Router DOM
- Ví dụ so sánh các lựa chọn UI stack:
  - Material UI
  - Radix UI
  - Shadcn UI
- Sử dụng TailwindCSS
- Nền tảng HTML accessibility
- Bài tập thực hành: nâng cấp bài form trước đó bằng UI libraries + TailwindCSS

### Tuần 7 - Tối Ưu Hiệu Năng I

- Nền tảng và trade-off giữa SSR vs CSR
- Áp dụng lazy loading và suspense
- Các pattern code splitting
- Dynamic imports
- Chiến lược named chunk
- Kỹ thuật debounce và throttle
- Bài tập thực hành: trang danh sách mới dùng lazy loading + debounce

### Tuần 8 - Tối Ưu Hiệu Năng II + Dự Án Cuối

- Chủ đề tối ưu React 18 (bao gồm nội dung liên quan hydration trong mô tả trang)
- Phân tích việc render của heavy components
- Phân tích Lighthouse treemap/bundle
- Tối ưu hình ảnh theo góc nhìn CDN
- Công cụ build được nhắc tới: Vite + Webpack/Rspack
- Dự án cuối: xây dựng web xử lý dữ liệu quy mô lớn (ví dụ nêu trên trang: ~100k records) và tối ưu bundle/performance

### Tuần 9 - Next.js Foundations

- Vì sao học Next.js sau React: tư duy full-stack, rendering pipeline, project structure
- Nền tảng App Router: segments, nested layouts, templates, route groups
- Server Components vs Client Components: boundaries, data flow, bundle impact
- Navigation primitives: `Link`, `useRouter`, loading UI, `error.tsx`, `not-found.tsx`
- Metadata, SEO, sitemap, robots, Open Graph
- Bài tập thực hành: xây dựng dashboard shell với nested layouts và route states

### Tuần 10 - Data Fetching và Rendering trong Next.js

- `fetch` trong Next.js, request caching, revalidation, cache tags
- Static, dynamic, ISR và partial pre-render trade-off
- Streaming và Suspense với App Router
- Server Actions cho mutation và form handling
- Route Handlers và ranh giới API
- Bài tập thực hành: dashboard dữ liệu với cached reads và write flows

### Tuần 11 - Auth, Database và System Flows

- Authentication và sessions với Auth.js trong App Router
- Tích hợp PostgreSQL + Prisma
- Validation và safe server boundaries với Zod
- File uploads, object storage và signed URL patterns
- Background jobs, cron, webhooks và async workflows
- Bài tập thực hành: mô-đun SaaS có auth, roles, data access và async tasks

### Tuần 12 - Production Architecture với Next.js

- Performance tuning trong Next.js: bundle analysis, RSC boundaries, image/font optimization
- Testing strategy: unit, integration và E2E cho App Router apps
- Observability: logging, tracing, error tracking, metrics
- Deployment trên Vercel: environments, secrets, preview deployments, rollback mindset
- Edge vs Node runtime trade-off và khi nào nên dùng mỗi loại
- Final capstone: production architecture cho multi-system app hỗ trợ SaaS, content và commerce
