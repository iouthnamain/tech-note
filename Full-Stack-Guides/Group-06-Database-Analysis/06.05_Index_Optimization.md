## Introduction / Giới thiệu

### Overview / Tổng quan

**English**: Indexes speed up queries but have costs. Understanding when and how to use indexes is crucial for database performance.

**Vietnamese**: Index tăng tốc truy vấn nhưng có chi phí. Hiểu khi nào và cách sử dụng index rất quan trọng cho hiệu năng database.

## Index Types / Loại index

### Example 1: Index Types / Ví dụ 1: Loại index

```sql
-- Primary Index (automatically created) / Index chính (tự động tạo)
CREATE TABLE users (
  id UUID PRIMARY KEY -- Creates primary index / Tạo index chính
);

-- Secondary Index / Index phụ
CREATE INDEX idx_user_email ON users(email);

-- Composite Index / Index tổng hợp
CREATE INDEX idx_user_name_email ON users(name, email);

-- Unique Index / Index duy nhất
CREATE UNIQUE INDEX idx_user_email_unique ON users(email);

-- Partial Index (PostgreSQL) / Index một phần
CREATE INDEX idx_active_users ON users(email) WHERE active = true;
```

## Index Best Practices / Thực hành tốt nhất cho index

### Example 3: Index Guidelines / Ví dụ 3: Hướng dẫn index

```sql
-- ✅ Good: Index on foreign keys / Tốt: Index trên khóa ngoại
CREATE INDEX idx_orders_user_id ON orders(user_id);

-- ✅ Good: Index on frequently queried columns / Tốt: Index trên cột thường truy vấn
CREATE INDEX idx_products_category_id ON products(category_id);

-- ✅ Good: Composite index for multi-column queries / Tốt: Index tổng hợp cho truy vấn nhiều cột
CREATE INDEX idx_orders_user_date ON orders(user_id, created_at);

-- ❌ Bad: Too many indexes / Xấu: Quá nhiều index
-- Each index slows down INSERT/UPDATE / Mỗi index làm chậm INSERT/UPDATE

-- ❌ Bad: Index on low-selectivity columns / Xấu: Index trên cột độ chọn lọc thấp
CREATE INDEX idx_users_gender ON users(gender); -- Only 2-3 values / Chỉ 2-3 giá trị
```

---

## Best Practices / Thực hành tốt nhất

1. **Index foreign keys** - Improve join performance
2. **Index WHERE columns** - Frequently filtered columns
3. **Index ORDER BY columns** - When sorting is common
4. **Avoid over-indexing** - Balance read/write performance
5. **Monitor index usage** - Remove unused indexes

---

## Summary / Tóm tắt

### Key Takeaways / Điểm chính

- **Use indexes**: Foreign keys, frequently queried columns
- **Avoid indexes**: Low selectivity, frequently updated
- **Balance**: Read performance vs write cost
- **Monitor**: Track index usage
