# Team Management Workflows

This document describes common workflows for using the Teams folder structure.

## Daily Workflow

### As Team Lead

1. **Morning**
   - Open [[Unified-Dashboard|Unified Dashboard]] or [[Team-Lead-Dashboard|Team Lead Dashboard]]
   - Review pending code reviews
   - Check backlog items needing attention
   - Review upcoming one-on-ones

2. **During Day**
   - Conduct code reviews (use [[Templates/Code-Review-Template|Code Review Template]])
   - Have one-on-ones (use [[Templates/One-on-One-Template|One-on-One Template]])
   - Update backlog items
   - Document technical decisions (use [[Templates/ADR-Template|ADR Template]])

3. **End of Day**
   - Update task statuses
   - Document learnings
   - Plan next day

## Weekly Workflow

### Team Lead Activities

1. **Monday**
   - Review team backlog
   - Plan week's one-on-ones
   - Review sprint progress

2. **Mid-Week**
   - Conduct one-on-ones
   - Review code reviews
   - Check team health

3. **Friday**
   - Update team health metrics
   - Review week's accomplishments
   - Plan next week

## Monthly Workflow

### Team Lead Activities

1. **Performance Reviews**
   - Use [[Templates/Performance-Review-Template|Performance Review Template]]
   - Review development plans
   - Set goals for next period

2. **Team Retrospectives**
   - Use [[Templates/Team-Retro-Template|Team Retro Template]]
   - Review team health
   - Identify improvements

3. **Team Health Review**
   - Use [[Templates/Team-Health-Template|Team Health Template]]
   - Review metrics
   - Identify trends

## Backlog Management Workflow

1. **Adding Items**
   - Create backlog item using [[Templates/Backlog-Item-Template|Backlog Item Template]]
   - Link to project tasks if applicable
   - Set priority and story points

2. **Prioritizing**
   - Review backlog regularly
   - Update priorities based on team capacity
   - Assign items to team members

3. **Tracking**
   - Update status as items progress
   - Link to project tasks when implemented
   - Close items when done

## One-on-One Workflow

1. **Preparation**
   - Review previous one-on-one notes
   - Check member's recent work
   - Prepare topics and feedback

2. **During Meeting**
   - Use [[Templates/One-on-One-Template|One-on-One Template]] or [[Templates/Quick-Templates/Quick-1on1-Template|Quick Template]]
   - Document discussion
   - Set action items

3. **Follow-up**
   - Update action items
   - Link to development plan if needed
   - Schedule next meeting

## Code Review Workflow

1. **As Reviewer**
   - Use [[Templates/Code-Review-Template|Code Review Template]]
   - Complete review checklist
   - Provide constructive feedback
   - Track review time

2. **Tracking**
   - Update code review status
   - Link to project tasks
   - Track metrics

## Technical Decision Workflow

1. **Document Decision**
   - Use [[Templates/ADR-Template|ADR Template]]
   - Document context and rationale
   - List alternatives considered

2. **Review**
   - Get team input
   - Update status
   - Link to related decisions

## Incident Response Workflow

1. **During Incident**
   - Use [[Templates/Incident-Response-Template|Incident Response Template]]
   - Document timeline
   - Track actions taken

2. **Post-Incident**
   - Complete root cause analysis
   - Document lessons learned
   - Create action items

## Integration Workflow

### Linking Teams to Projects

1. **Team Projects**
   - Add projects to team frontmatter
   - Create project links in `Projects/` folder
   - Link backlog items to project tasks

2. **Member Links**
   - Link team members to project members
   - Track assignments across teams and projects

### Linking to Developer Workspace

1. **Code Reviews**
   - Link Team Lead code reviews to Developer code reviews received
   - Track learnings in Developer workspace

2. **Technical Notes**
   - Link technical decisions to Developer technical notes
   - Share knowledge between Team Lead and Developer roles

## Quick Reference

- **Quick Templates**: Use for fast note-taking during meetings
- **Comprehensive Templates**: Use for detailed documentation and formal reviews
- **Standard Templates**: Use for regular activities
