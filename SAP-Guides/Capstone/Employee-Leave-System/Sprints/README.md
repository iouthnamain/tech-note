# Sprint Documentation

This folder contains detailed documentation for each sprint of the Employee Leave Request and Approval System capstone project.

## Sprint Structure

Each sprint is documented in a separate markdown file following the naming convention:
- `Sprint01_Requirements_Gathering.md`
- `Sprint02_Detailed_Design.md`
- `Sprint03_Foundation.md`
- etc.

## Sprint Documentation Contents

Each sprint document includes:

1. **Sprint Overview** - Dates, goals, team capacity
2. **Sprint Backlog** - User stories with story points
3. **Daily Breakdown** - Tasks assigned to each team member by day
4. **Daily Standup Notes** - Template for tracking daily progress
5. **Sprint Review** - Summary of completed work
6. **Sprint Retrospective** - Lessons learned and improvements
7. **Burndown Tracking** - Progress tracking throughout the sprint
8. **Risk Log** - Identified risks and mitigation strategies
9. **Blockers & Dependencies** - Issues preventing progress
10. **Definition of Done Checklist** - Verification of completion
