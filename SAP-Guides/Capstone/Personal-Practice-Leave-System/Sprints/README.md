# Sprint Documentation

This folder contains detailed documentation for each sprint of the Employee Leave Request and Approval System personal practice project.

## Sprint Structure

Each sprint is documented in a separate markdown file following the naming convention:
- `Sprint01_Requirements_Gathering.md`
- `Sprint02_Detailed_Design.md`
- `Sprint03_Foundation.md`
- etc.

## Sprint Documentation Contents

Each sprint document includes:

1. **Sprint Overview** - Dates, goals, solo developer capacity
2. **Sprint Backlog** - User stories with story points (adjusted for solo)
3. **Daily Breakdown** - Tasks for each day
4. **Self-Review Checklist** - Template for tracking daily progress
5. **Sprint Review** - Summary of completed work
6. **Sprint Retrospective** - Lessons learned and improvements
7. **Progress Tracking** - Progress tracking throughout the sprint
8. **Learning Notes** - Key learnings from the sprint
