# Developer Workflows

This page defines how the `Developer` area should be used with `mds` as the control center.

## Weekly Workflow

1. Open [Career OS Dashboard](./mds/00-Dashboard.md).
2. Follow the current phase from [Study Roadmap](./mds/01-Study-Roadmap.md).
3. Build or update one project page in [Projects](./mds/05-Projects/README.md).
4. Record technical lessons in [Technical Notes](./Technical-Notes/Technical-Notes.md) when they are worth keeping.
5. Finish the week in [Weekly Reviews](./mds/06-Weekly-Reviews/README.md).

## Coding Workflow

1. Start from [Coding Approach](./mds/02-Coding-Approach.md).
2. Build the smallest useful version first.
3. Save reusable patterns in [Code Snippets](./Code-Snippets/Code-Snippets.md).
4. Save review lessons in [Code Reviews](./Code-Reviews/Code-Reviews.md).

## Profile Workflow

1. Record project evidence in [Profile Builder](./mds/03-Profile-Builder.md).
2. Convert evidence into GitHub, LinkedIn, and resume inputs.
3. Update profile material monthly, not randomly.

## Leadership Workflow

1. Practice self-management and task planning first.
2. Improve progress updates, handoffs, and review quality.
3. Add mentoring and coordination only after shipping real work consistently.
