# Profile Builder

Build your profile from evidence, not from vague claims.

## Professional Summary

Write one short English summary:

> Frontend-first software engineer in Vietnam building toward full-stack delivery. I focus on practical product work, clean execution, and improving through real projects, code review, and weekly iteration.

Adjust this only when your real work changes.

## Skills Inventory

Split skills into three levels:

| Strong | Learning | Exposure |
| --- | --- | --- |
| HTML, CSS, Git | JavaScript, React, API integration | Node.js, PostgreSQL, Docker |

Rules:

- `Strong` means you can explain and use it without help.
- `Learning` means you are actively building with it now.
- `Exposure` means you touched it, but should not sell it aggressively yet.

## Project Evidence Table

Use one row per project or meaningful feature:

| Project | Problem | Stack | Responsibility | Result | Lessons |
| --- | --- | --- | --- | --- | --- |
| Project 01 | Need a small app to practice frontend + API flow | HTML, CSS, JS, API | UI, form flow, validation, notes | Built and documented a working MVP | Learned request flow and error handling |

Only keep evidence you can discuss clearly in an interview.

## Achievement Log

Track measurable outcomes:

| Date | Achievement | Evidence | Why It Matters |
| --- | --- | --- | --- |
| 2026-03-23 | Created Career OS workspace | `Developer/mds` | Built a repeatable learning and profile system |

Use the detailed template in [Achievement Log Template](./Templates/Achievement-Log-Template.md).

## GitHub Checklist

- pin projects that match your target role
- write clear README files with setup, screenshots, and decisions
- keep commit messages readable
- show issues, task notes, or milestones when possible
- remove dead or confusing demo repos from pinned slots
- prefer fewer better repos over many weak repos

## LinkedIn And Resume Inputs

Write these from project evidence:

- short headline
- professional summary
- skills list
- 3-5 strongest project bullets
- measurable outcomes
- collaboration examples

Avoid:

- vague claims like `hardworking` or `passionate`
- listing tools you cannot explain
- copying AI-written achievements that you did not earn

## Monthly Update Routine

At the end of each month:

1. Update the skills inventory.
2. Add new project evidence.
3. Rewrite weak bullets using clearer outcomes.
4. Check whether GitHub and resume still reflect your real level.
