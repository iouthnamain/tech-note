# Exercise 1: Build a Semantic HTML5 Page

## Objective
Create a well-structured HTML5 page using semantic elements and proper form validation.

## Requirements

### Part 1: Semantic Structure

Create an HTML5 page with:
1. **Header** - Site header with navigation
2. **Main** - Main content area
3. **Article** - At least one article with sections
4. **Aside** - Sidebar with related content
5. **Footer** - Site footer with contact information

### Part 2: Form with Validation

Create a registration form with:
1. **Name field** - Text input, required
2. **Email field** - Email input, required
3. **Password field** - Password input, min 8 characters
4. **Age field** - Number input, range 18-100
5. **Country dropdown** - Select with at least 5 options
6. **Newsletter checkbox** - Optional subscription
7. **Submit button** - Proper form submission

### Part 3: Accessibility

Add ARIA attributes:
- Proper labels for all form fields
- ARIA labels for buttons
- Semantic HTML5 elements
- Proper heading hierarchy (h1, h2, h3)

### Part 4: HTML5 Features

Use HTML5 features:
- Semantic elements (header, nav, article, section, aside, footer)
- New input types (email, number, date)
- Form validation attributes (required, min, max, pattern)
- Time element for dates

## Starter Code

Create `index.html`:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Your Page Title</title>
</head>
<body>
    <!-- Your semantic HTML structure here -->
</body>
</html>
```

## Expected Structure

```
Header
  - Navigation
Main
  - Article
    - Section 1
    - Section 2
  - Aside
    - Related content
  - Form section
    - Registration form
Footer
  - Contact info
```

## Testing

1. Validate HTML using [W3C Validator](https://validator.w3.org/)
2. Test form validation in browser
3. Check accessibility with screen reader
4. Test responsive design

## Bonus Challenges

1. Add a search form in the header
2. Create a multi-step form
3. Add data attributes for JavaScript
4. Implement form with JavaScript validation
5. Add microdata or JSON-LD for SEO
